O que é um INSERT?

O comando INSERT INTO, que significa "inserir", é usado para incluir informações em uma tabela do BD (banco de dados). Geralmente nos referimos ao comando como "INSERT" mas a sintaxe correta é INSERT INTO seguido do nome da tabela e os valores desejados conforme o exemplo.

Sintaxe de um INSERT INTO em SQL
INSERT INTO nome_da_tabela (nome_da_coluna1,nome_da_coluna2, ...)
VALUES ('valor1', 'valor2', ...);

INSERT INTO nome_da_tabela
VALUES ('valor1', 'valor2', ...);

No exemplo abaixo a tabela se chama CLIENTES e as colunas que serão inseridas sao "nomeCliente" e "cidadeCliente". Antes de executar o INSERT, faça um SELECT na tabela CLIENTES e verifique os campos que ela possui.

INSERT INTO CLIENTES  (nomeCliente, cidadeCliente)
VALUES ('Maria José', 'Brasília');

Após executar o INSERT, faça outra vez o SELECT e perceba que o campo idCliente será preenchido, mesmo não incluindo este campo na Query. Isso ocorre pois o campo idCliente é do tipo INT AUTO_INCREMENT ou seja, é um número inteiro que será preenchido automaticamente, adicionando +1 ao último número registrado. O primeiro registro recebe o número 1, o segundo recebe o número 2, e assim por diante. Caso seja passado um ID, conforme exemplo abaixo, também irá funcionar, sem seguir a sequência.

INSERT INTO CLIENTES  (idCliente, nomeCliente, cidadeCliente)
VALUES (20,'Maria José', 'Brasília');

Foram mostradas 2 sintaxes diferentes para o INSERT. A primeira com os nomes dos campos e valores. A segunda sem os nomes dos campos. Isso é possível se você inserir os valores na ordem certa em todos os campos da tabela, exceto o campo configurado com AUTO_INCREMENT (geralmente é o id), pois ele será preenchido automaticamente em ordem crescente.

Alguns conceitos importantes

AUTO_INCREMENT: É a sintaxe do MySQL para que uma coluna gere um número automatico, incrementando de 1 em 1, mas para outros BDs a sintaxe e a maneira de fazer isso pode variar, como no SQL SERVER, usa-se o IDENTITY, já no ORACLE, não existe uma palavra-chave para essa configuração.

PRIMARY KEY: Se quisermos escolher o valor do ID na hora de fazer o INSERT, dará certo. Mas se o campo ID for uma PRIMARY KEY (chave primária), este campo não poderá ser repetido. Existem várias formas de não permitir valores repetidos em uma coluna de uma tabela. Isso varia de acordo com cada BD. Bem como esse não é o único objetivo de criar uma PRIMARY KEY. A chave primária é o que identifica um único registro em uma tabela e ela pode ser composta por mais de uma coluna!
