Como usar IS NULL e IS NOT NULL?

O comando IS NULL, que significa "é nulo", é usado para validar se um campo da tabela tem valor NULL. O comando IS NOT NULL faz o oposto.

Sintaxe IS NULL
SELECT nome_da_coluna, nome_da_coluna2, ...
FROM nome_da_tabela WHERE nome_da_coluna IS NULL;
Sintaxe IS NULL
SELECT nome_da_coluna, nome_da_coluna2, ...
FROM nome_da_tabela WHERE nome_da_coluna IS NOT NULL;

No exemplo abaixo vamos trazer os clientes onde a UF tem o valor NULL, e em seguida, onde a UF não é NULL.

SELECT * FROM CLIENTES
WHERE UFCliente IS NULL;

SELECT * FROM CLIENTES
WHERE UFCliente IS NOT NULL;

Note que não podemos usar os operadores de comparação (=, <, >, etc...) para testar valores nulos. Por exemplo se escrevermos "WHERE nomeCliente = NULL" não funcionará.

O operador IS NULL é equivalente a "igual a NULL" e o IS NOT NULL é equivalente a "diferente de NULL".

Alguns conceitos importantes

Valor NULL: Você deve ter percebido que o SELECT exibe a palavra NULL em algum momento. Quando não há valor em uma coluna específica de um registro, é exibido como "NULL" que significa "nulo". Ao criar um campo podemos defini-lo como NOT NULL, ou seja, o campo torna-se obrigatório pois a tabela não irá aceitar que o mesmo seja vazio em algum registro.
