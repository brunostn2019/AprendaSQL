O que é o WHERE?

A cláusula WHERE, que significa "onde", é usado para criar filtros limitando a quantidade de registros que serão afetados por uma instrução SQL. Quando utilizado, o WHERE cria uma condição necessária para que a ação desejada aconteça.

Sintaxe de um WHERE em SQL
SELECT nome_da_coluna, nome_da_coluna2, ...
FROM nome_da_tabela
WHERE condição_desejada;

No primeiro exemplo abaixo queremos trazer apenas os clientes que tem o id = 1. Então é como se dissessemos "selecione os clientes ONDE o id é igual a 1". No segundo exemplo não deve retornar nada pois não existe um cliente com id = 100. No terceiro exemplo, deve retornar todos os clientes que moram em MG.

SELECT *  FROM CLIENTES  WHERE idCliente = 1;
SELECT *  FROM CLIENTES  WHERE idCliente = 100;
SELECT *  FROM CLIENTES  WHERE UFCliente = 'MG';

As condições podem usar outros operadores de comparação, igual (=), menor que (<),maior que (>), entre outros.

Alguns conceitos importantes

Aspas: Indica o inicio e o fim de uma STRING em SQL. Você pode usar "aspas duplas" ou 'aspas simples' em suas Queries, mas na maioria dos BDs usamos aspas simples. Quando comparamos números, não utilizamos aspas, conforme exemplos da Tabela de Operadores.

Campos do tipo DATA (DATE): As datas são representadas no formato 'aaaa-MM-dd' ou 'yyyy-MM-dd' sendo 'a' de ano ('y' de year), 'M' de mes (month), 'd' de dia (day). Também usa-se aspas nos valores do tipo DATA.

Campos do tipo DATA e HORA (DATETIME): Semelhante ao tipo DATE, mas usa-se 'aaaa-MM-dd HH:MI:SS' onde 'H' é a hora, 'MI' são minutos e 'SS' os segundos.

Outros tipos de DATA: Também existem tipos diferentes relacionados a data, e podem variar de acordo com o BD utilizado, por exemplo, em MySQL existem os tipos TIMESTAMP e o YEAR. Use o GOOGLE sempre que precisar para saber exatamente qual formato utilizar sem seu BD.
