SELECT DISTINCT

O comando SELECT DISTINCT, que significa "selecionar distinto", é usado para buscar informações diferentes (distintas) em uma tabela do BD (banco de dados). Alguns registros terão campos com valores iguais, como o campo cidadeCliente, ou UFCliente, na nossa tabela CLIENTES.

Sintaxe de um SELECT DISTINCT em SQL
SELECT DISTINCT nome_da_coluna, nome_da_coluna2, ...
FROM nome_da_tabela;

No exemplo abaixo a tabela se chama "CLIENTES" e as colunas que serão exibidas são "cidadeCliente" e "UFCliente". Faça o mesmo comando sem usar a palavra DISTINCT para ver a diferença nos resultados.

SELECT DISTINCT cidadeCliente, UFCliente
FROM CLIENTES;

O DISTINCT só funciona quando todas as colunas solicitadas tem valores repetidos, se em 2 registros uma das colunas for diferente, e todas as outras forem iguais, será exibido como 2 registros. Se todas as colunas forem iguais, será exibido como 1 registro.

Quando precisarei usar o distinct? Muitas vezes! Suponha que queremos saber quantas, ou quais, cidades diferentes existem entre os clientes de nossa tabela, faça um select DISTINCT apenas no campo "cidadeCliente".

Alguns conceitos importantes

Case Sensitive: é quando precisamos diferenciar letras maiúsculas de minúsculas para que nosso código funcione. As palavras-chave em SQL NÃO é Case Sensitive. Observe que select é igual a SELECT.

SQL Data Types: Data Types significa "tipos de dados". Existem muitos tipos de dados para serem usados em nossos CAMPOS das tabelas. Quando uma tabela é criada deve-se dizer qual o nome da coluna e qual o tipo de dado dessa coluna, além de outras informações que veremos aos poucos. Os tipo de dados podem ser variações do tipo STRING (baseados em texto), NUMÉRICOS (inteiros, decimais, etc), DATE and TIME (Baseados em data e hora / tempo) entre outros.

Valor NULL: Você deve ter percebido que o SELECT exibe a palavra NULL em algum momento. Quando não há valor em uma coluna específica de um registro, é exibido como "NULL" que significa "nulo". Ao criar um campo podemos defini-lo como NOT NULL, ou seja, o campo torna-se obrigatório pois a tabela não irá aceitar que o mesmo seja vazio em algum registro.

Comentários: Ao escrever códigos em SQL podemos criar comentários entre as linhas de código, essa parte não é executada e não causa erros no código. Isso é muito utilizado quando queremos explicar o que pretendemos fazer em determinada parte do código e facilita para que outras pessoas possam ler o código e entender mais claramente no futuro, se necessário. Para criar um comentário de uma linha, coloca-se '-- ' (sem as aspas) antes da linha. Para um comentário de várias linhas, o texto deve estar entre os caracteres '/*' e '*/' sem as aspas. Exemplo: /* texto do comentario em várias linhas */. O editor do APRENDA SQL não suporta comentários no momento.
